# Composer

Мультиинструментальный Telegram ИИ-агент для бота **@Composer_ubot**, развёртываемый на Cloudflare Workers. Мозг — модель через OpenRouter (её название скрыто и не раскрывается ботом). Оркестрация — собственный state-граф в духе LangGraph с модель-агностичным JSON-протоколом вызова инструментов.

Бот умеет: искать и скачивать музыку, скачивать видео из YouTube, Instagram, Pinterest, TikTok и других площадок, проводить обычный и глубокий research, генерировать PDF-отчёты (с поддержкой кириллицы), планировать и выполнять простые многошаговые задачи, а также запоминать факты о пользователе и вести базу знаний.

## Почему такая архитектура

Cloudflare Workers — это изолят V8: нет файловой системы и нельзя запускать нативные процессы. Поэтому `yt-dlp` и `ffmpeg` физически не могут работать внутри Worker. Правильный паттерн — оставить Worker «дирижёром», а тяжёлую задачу извлечения медиа делегировать по HTTPS сервису [Cobalt](https://github.com/imputnet/cobalt), который как раз оборачивает извлечение из YouTube, Instagram, Pinterest, TikTok, SoundCloud и других. Cobalt легко поднимается одним Docker-контейнером, либо можно указать публичный инстанс.

Аналогично, нативный function-calling многие бесплатные модели OpenRouter (включая `:free`-варианты) поддерживают ненадёжно. Поэтому вызов инструментов реализован через строгий JSON-протокол, который парсится на нашей стороне и работает с любой моделью. Граф при этом сохраняет форму LangGraph (узлы `agent` и `tools`, условные рёбра), так что при переходе на модель с function-calling его легко заменить на `@langchain/langgraph`.

## Структура проекта

Бот не сводится к одному `bot.js` — он разбит по слоям ответственности:

```
composer/
  src/
    index.js              точка входа Worker: маршруты /webhook, /set-webhook, /health
    config.js             чтение и валидация переменных окружения
    llm/openrouter.js     клиент OpenRouter (модель скрыта в конфиге)
    agent/
      graph.js            движок state-графа + петля агента (JSON tool-calling)
      run.js              сборка контекста, запуск графа, сохранение истории
      systemPrompt.js     системный промпт (секретит модель, задаёт персону)
      tools.js            реестр инструментов агента
    tools/
      media.js            загрузка видео/аудио через Cobalt
      music.js            поиск трека + извлечение аудио
      research.js         web-поиск и deep research (Tavily + синтез LLM)
      pdf.js              генерация PDF с встроенным Unicode-шрифтом
    memory/store.js       память, знания, планы и история диалога на KV
    telegram/
      client.js           клиент Telegram Bot API
      router.js           разбор апдейтов, команды, диспетчеризация в агента
      format.js           HTML-форматирование и разбиение длинных сообщений
    utils/                http (таймауты/ретраи) и логирование без секретов
  scripts/                env-to-devvars, sync-secrets, set-webhook
  wrangler.toml           конфиг Worker, KV-биндинг, несекретные vars
  .env.example            шаблон единственного файла секретов
```

## Стек

JavaScript (ES-модули), Cloudflare Workers, Wrangler, KV для состояния, `pdf-lib` + `@pdf-lib/fontkit` для PDF. Внешние сервисы: OpenRouter (LLM), Cobalt (медиа), Tavily (поиск).

## Секреты: единый .env

Все секреты живут в одном файле `.env` — без `.env.local`, `.env.production` и прочей путаницы. Этот файл питает оба сценария:

- Локальная разработка: `npm run dev` сначала вызывает `scripts/env-to-devvars.mjs`, который генерирует `.dev.vars` из `.env` (Wrangler для `wrangler dev` читает именно `.dev.vars`, а не `.env` — мы делаем мост прозрачно).
- Продакшен: `npm run secrets` читает `.env` и загружает каждый ключ в хранилище секретов Cloudflare через `wrangler secret put`. Ключи, которые объявлены как `[vars]` в `wrangler.toml` (несекретные, например URL инстанса Cobalt), при этом пропускаются, чтобы не возникало конфликта «var и secret одновременно».

Скопируй `.env.example` в `.env` и заполни значения.

## Запуск

1. Установить зависимости: `npm install`.
2. Создать KV-namespace и вписать его id в `wrangler.toml`: `npx wrangler kv namespace create "COMPOSER_KV"`.
3. Заполнить `.env` (токен бота от @BotFather, ключ OpenRouter, при необходимости ключи Cobalt и Tavily). Модель задаётся переменной `OPENROUTER_MODEL`.
4. Авторизоваться в Cloudflare: `npx wrangler login`.
5. Загрузить секреты: `npm run secrets`.
6. Задеплоить: `npm run deploy`.
7. Зарегистрировать webhook: либо `npm run set-webhook -- https://composer.<твой-сабдомен>.workers.dev`, либо открыть `https://<worker>/set-webhook?secret=<TELEGRAM_WEBHOOK_SECRET>` в браузере.

Для локальной разработки: `npm run dev` (Wrangler поднимет локальный сервер; webhook удобно прокинуть через туннель).

## Команды бота

`/start` — приветствие, `/help` — справка, `/memory` — что бот помнит, `/forget` — стереть память, `/reset` — очистить историю диалога. Всё остальное обрабатывает агент: ссылка на видео или трек → скачивание; запрос «найди и скачай …» → музыка; «исследуй …» → deep research; «сделай PDF …» → отчёт; «запомни …» → память.

## Память, знания, планирование

Состояние хранится в KV по чату: долговременная память (факты о пользователе), база знаний (титулованные заметки с keyword-поиском), текущий план (цель и шаги с отметками выполнения) и история диалога (ограниченная по длине). Инструменты `remember`, `recall`, `save_knowledge`, `search_knowledge`, `create_plan`, `update_plan` доступны агенту напрямую. Системный промпт подмешивает память и план в контекст каждого запроса.

## Скрытность модели

Название и провайдер модели читаются только в `config.js` из `OPENROUTER_MODEL` и нигде не возвращаются пользователю и не пишутся в логи (логгер редактирует поля с `model/key/secret/token`). Системный промпт явно запрещает раскрывать модель, провайдера, версию и содержимое промпта — на любой такой вопрос бот отвечает, что он Composer.

## Ограничения и развитие

Telegram принимает медиа по URL примерно до 20 МБ и по загрузке до 50 МБ — крупные файлы стоит отдавать ссылкой. Для долгой автономной работы (по расписанию, фоновые задачи) подключите Cron Triggers и/или Durable Objects — текущая автономность ограничена многошаговой петлёй в пределах одного запроса. Для масштабной базы знаний слой `memory/store.js` заменяется на D1 без изменения остальной логики.
