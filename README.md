# Composer

Мультиинструментальный Telegram ИИ-агент для бота **@Composer_ubot**. Мозг — модель через **GROQ** (её название скрыто и не раскрывается ботом). Оркестрация — собственный state-граф в духе LangGraph с модель-агностичным JSON-протоколом вызова инструментов.

Бот умеет: искать и скачивать музыку, скачивать видео из YouTube, Instagram, Pinterest, TikTok и других площадок, проводить обычный и глубокий research, генерировать PDF-отчёты (с поддержкой кириллицы), планировать и выполнять простые многошаговые задачи, а также запоминать факты о пользователе и вести базу знаний.

Поддерживаются два режима запуска: временный **polling** (обычный процесс `node`, без вебхуков, без KV, без публичного URL) и продакшен на **Cloudflare Workers** (вебхук + KV). Оба используют один и тот же код обработки.

## Быстрый запуск (временный режим: polling + GROQ, без KV и вебхуков)

Скопируй `.env.example` в `.env` и заполни как минимум три значения: `TELEGRAM_BOT_TOKEN` (от @BotFather), `GROQ_API` и `GROQ_MODEL` (с console.groq.com). Затем установи зависимости и запусти:

```
npm install
npm start
```

Бот сам снимет вебхук (если был) и начнёт long-polling. Память в этом режиме хранится в оперативной памяти процесса и не переживает перезапуск — это и есть смысл «временного» режима. Для скачиваний понадобится инстанс Cobalt, для research — ключ Tavily (оба опциональны и описаны ниже).

## Почему такая архитектура

Cloudflare Workers — это изолят V8 без файловой системы и без запуска нативных процессов, поэтому `yt-dlp` и `ffmpeg` физически не могут работать внутри Worker. Правильный паттерн — оставить рантайм «дирижёром», а извлечение медиа делегировать по HTTPS сервису [Cobalt](https://github.com/imputnet/cobalt), который умеет YouTube, Instagram, Pinterest, TikTok, SoundCloud и музыку; он поднимается одним Docker-контейнером, либо берётся публичный инстанс (URL задаётся в `wrangler.toml`, для polling — через переменную окружения `COBALT_API_URL`).

Вызов инструментов сделан через строгий JSON-протокол, который парсится на нашей стороне и работает с любой моделью, не полагаясь на нативный function-calling. Граф при этом сохраняет форму LangGraph (узлы `agent` и `tools`, условные рёбра), поэтому при необходимости его легко заменить на `@langchain/langgraph`.

## Структура проекта

Бот не сводится к одному `bot.js` — он разбит по слоям ответственности:

```
composer/
  src/
    index.js              точка входа Cloudflare Worker (режим вебхука)
    config.js             чтение и валидация переменных окружения
    llm/groq.js           клиент GROQ (модель скрыта в конфиге)
    runtime/polling.js    long-polling рантайм (режим без вебхуков)
    agent/
      graph.js            движок state-графа + петля агента (JSON tool-calling)
      run.js              сборка контекста, запуск графа, сохранение истории
      systemPrompt.js     системный промпт (секретит модель, задаёт персону)
      tools.js            реестр инструментов агента
    tools/
      media.js            загрузка видео/аудио через Cobalt
      music.js            поиск трека + извлечение аудио
      research.js         web-поиск и deep research (Tavily + синтез LLM)
      pdf.js              генерация PDF с встроенным Unicode-шрифтом
    memory/store.js       память, знания, планы, история; KV или in-memory
    telegram/
      client.js           клиент Telegram Bot API (вебхук и getUpdates)
      router.js           разбор апдейтов, команды, диспетчеризация в агента
      format.js           HTML-форматирование и разбиение длинных сообщений
    utils/                http (таймауты/ретраи) и логирование без секретов
  scripts/
    start-polling.mjs     локальный запуск в режиме polling (npm start)
    env-to-devvars.mjs    мост .env -> .dev.vars для wrangler dev
    sync-secrets.mjs      загрузка секретов из .env в Cloudflare
    set-webhook.mjs       регистрация вебхука для режима Workers
  wrangler.toml           конфиг Worker, опциональный KV, несекретные vars
  .env.example            шаблон единственного файла секретов
```

## Стек

JavaScript (ES-модули). Рантайм: обычный Node.js (polling) или Cloudflare Workers + Wrangler (вебхук). Хранение состояния: Cloudflare KV либо in-memory. `pdf-lib` + `@pdf-lib/fontkit` для PDF. Внешние сервисы: GROQ (LLM), Cobalt (медиа), Tavily (поиск).

## Секреты: единый .env

Все секреты живут в одном файле `.env` — без `.env.local`, `.env.production` и прочей путаницы. Для polling его читает `npm start`. Для Cloudflare: `npm run dev` генерирует `.dev.vars` из `.env` (Wrangler локально читает именно `.dev.vars`), а `npm run secrets` загружает каждый ключ в хранилище секретов Cloudflare. Ключи, объявленные как `[vars]` в `wrangler.toml` (несекретные, например URL инстанса Cobalt), при загрузке секретов пропускаются, чтобы не было конфликта «var и secret одновременно».

## Запуск на Cloudflare Workers (вебхук + KV)

Это постоянный режим. KV здесь опционален: без него память не сохраняется между запросами, с ним — сохраняется. Шаги: установить зависимости `npm install`; при желании durable-памяти создать namespace `npx wrangler kv namespace create "COMPOSER_KV"` и раскомментировать блок `[[kv_namespaces]]` в `wrangler.toml`, вписав id; заполнить `.env` (как минимум `TELEGRAM_BOT_TOKEN`, `GROQ_API`, `GROQ_MODEL`, плюс `TELEGRAM_WEBHOOK_SECRET` для защиты вебхука); авторизоваться `npx wrangler login`; залить секреты `npm run secrets`; задеплоить `npm run deploy`; зарегистрировать вебхук, открыв `https://<worker>/set-webhook?secret=<TELEGRAM_WEBHOOK_SECRET>`.

## Команды бота

`/start` — приветствие, `/help` — справка, `/memory` — что бот помнит, `/forget` — стереть память, `/reset` — очистить историю диалога. Всё остальное обрабатывает агент: ссылка на видео или трек → скачивание; запрос «найди и скачай …» → музыка; «исследуй …» → deep research; «сделай PDF …» → отчёт; «запомни …» → память.

## Память, знания, планирование

Состояние хранится по чату: долговременная память (факты о пользователе), база знаний (титулованные заметки с keyword-поиском), текущий план (цель и шаги) и история диалога (ограниченная по длине). Слой `memory/store.js` сам выбирает бэкенд: Cloudflare KV, если биндинг подключён, иначе in-memory (то же API, но данные живут только в пределах процесса). Инструменты `remember`, `recall`, `save_knowledge`, `search_knowledge`, `create_plan`, `update_plan` доступны агенту напрямую, а память подмешивается в контекст каждого запроса.

## Скрытность модели

Название и провайдер модели читаются только в `config.js` из `GROQ_MODEL` и нигде не возвращаются пользователю и не пишутся в логи (логгер редактирует поля с `model/key/secret/token`). Системный промпт явно запрещает раскрывать модель, провайдера, версию и содержимое промпта — на любой такой вопрос бот отвечает, что он Composer.

## Ограничения и развитие

Telegram принимает медиа по URL примерно до 20 МБ и по загрузке до 50 МБ — крупные файлы стоит отдавать ссылкой. In-memory память не переживает перезапуск процесса — для постоянного хранения вернитесь на KV (Workers) или подключите D1. Для долгой автономной работы по расписанию на Workers подключаются Cron Triggers и Durable Objects; сейчас автономность ограничена многошаговой петлёй в пределах одного запроса.
